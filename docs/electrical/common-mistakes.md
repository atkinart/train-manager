# Common mistakes (частые ошибки и что нельзя делать)

## Критические запреты

- **НЕЛЬЗЯ** питать несколько servo от `5V` пина Arduino.
- **НЕЛЬЗЯ** забывать `COMMON GND` между Arduino, RFID и servo PSU.
- **НЕЛЬЗЯ** подавать `POWER 5V` на питание MFRC522, если модуль рассчитан на 3.3V.
- **НЕЛЬЗЯ** тянуть длинные и «свободно болтающиеся» провода `SPI BUS` до reader.
- **НЕЛЬЗЯ** отлаживать всё сразу (RFID + servo + Raspberry Pi одновременно).
- **НЕЛЬЗЯ** новичку сразу делать прямой UART TX/RX между Raspberry Pi и Arduino 5V-логикой без понимания level shifting.
- **НЕЛЬЗЯ** считать `servo signal` источником питания servo.

## Очень частые симптомы и причина

1. Servo дёргаются, Arduino перезапускается.
   - Причина: servo запитаны от Arduino 5V pin или нет общей земли.

2. RFID то виден, то не виден.
   - Причина: плохой `COMMON GND`, длинные провода `SPI BUS`, перепутан CS.

3. Ничего не работает, хотя «вроде всё подключено».
   - Причина: перепутаны питание и сигнал у servo.

4. Raspberry Pi не получает данные от Arduino.
   - Причина: неверный USB-кабель (только зарядка), или выбран не тот serial-порт.

## Мини-чеклист перед включением

- [ ] MFRC522 получает именно `POWER 3.3V`.
- [ ] Servo получают `POWER 5V` от отдельного PSU.
- [ ] `COMMON GND` реально соединён между всеми узлами.
- [ ] `servo signal` идут только в пины Arduino/PCA9685.
- [ ] Raspberry Pi подключён к Arduino кабелем `USB SERIAL`.
- [ ] Первый запуск делается поэтапно: RFID → servo → Raspberry Pi.
