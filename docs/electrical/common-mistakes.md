# Common Mistakes (и как избежать)

## Ошибка 1: подать 5V на GPIO Pico

**Почему плохо:** риск повреждения микроконтроллера.

**Как правильно:** все сигнальные линии Pico должны быть 3.3V.

## Ошибка 2: питать servo от Pico

**Почему плохо:** пиковый ток servo вызывает reset/нестабильность.

**Как правильно:** внешний 5V PSU для servo, но с общим GND.

## Ошибка 3: забыть common ground

**Почему плохо:** сигнал PWM и SPI становится «плавающим».

**Как правильно:** объединить GND Pico, MFRC522 и servo PSU.

## Ошибка 4: использовать USB кабель без data-линий

**Почему плохо:** Pico не виден как serial device.

**Как правильно:** брать data-USB кабель.

## Ошибка 5: отлаживать всё одновременно

**Почему плохо:** сложно понять источник ошибки.

**Как правильно:** RFID -> Servo -> Serial -> MQTT -> Backend -> UI.
