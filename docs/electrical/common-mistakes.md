# Common mistakes

## Критические ошибки

- Подача 5V сигнала на GPIO Pico.
- Питание servo от Pico.
- Отсутствие COMMON GND.
- Попытка сразу включить RFID + servo + bridge + backend без поэтапной проверки.

## Симптомы и причины

1. Servo дёргаются, Pico перезагружается.
   - Причина: ошибка питания servo или нет общей земли.
2. RFID работает нестабильно.
   - Причина: длинные SPI провода, неверные пины, плохой GND.
3. Нет serial данных на Raspberry Pi.
   - Причина: charge-only USB кабель или неверный serial port.

## Чеклист перед первым запуском

- [ ] MFRC522 питается только от 3.3V.
- [ ] Servo питаются от отдельного 5V PSU.
- [ ] Все земли объединены.
- [ ] Pico подключён к Raspberry Pi data USB-кабелем.
