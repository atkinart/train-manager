# Arduino ↔ Raspberry Pi connection

## Почему USB Serial — лучший старт

Для новичка самый безопасный и простой вариант — **USB Serial**:

- один кабель даёт и связь, и питание Arduino,
- не нужно разбираться с уровнями TX/RX,
- меньше шансов повредить GPIO Raspberry Pi,
- проще дебажить (`/dev/ttyACM*` или `/dev/ttyUSB*`).

## Что физически соединяется

- USB-порт Raspberry Pi (обычно USB-A)
- USB-порт Arduino (USB-C/USB-B в зависимости от платы)
- data-кабель USB (не «только зарядка»)

## Схема

См. `diagrams/arduino-rpi-serial.svg`.

На схеме явно выделены:
- `USB SERIAL`
- отдельное питание Raspberry Pi
- предупреждение про прямой UART

## Пошагово

1. Выключите питание узла.
2. Подключите USB-кабель Raspberry Pi ↔ Arduino.
3. Включите Raspberry Pi.
4. Убедитесь, что в системе появился serial-порт Arduino.
5. Только после этого запускайте обмен командами.

## Что можно позже

Позже возможны варианты:
- UART (TX/RX) с правильным согласованием уровней,
- Ethernet/Wi‑Fi мост,
- несколько Arduino node через MQTT-шлюз.

Но для MVP это не нужно.

## Почему прямой UART TX/RX не нужен новичку

Потому что у прямого подключения больше рисков:
- можно ошибиться с уровнями 5V/3.3V,
- можно перепутать TX/RX/GND,
- сложнее первичная диагностика.

**Рекомендация:** на старте только `USB SERIAL`.
