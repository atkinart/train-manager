# Power design (архитектура питания)

## Главное правило

В проекте есть **разные зоны питания**:

- `POWER 5V` для servo (силовая часть)
- `POWER 3.3V` для MFRC522
- питание Arduino (обычно через USB)
- питание Raspberry Pi (своим блоком)

Их можно питать от разных источников, **но земля должна быть общей**: `COMMON GND`.

## Что такое GND и почему это важно

`GND` — это общий «нулевой» уровень.

Сигналы (`SPI`, `servo signal`) измеряются относительно GND. Если GND не общий:
- сигналы становятся «непонятными» для устройств,
- RFID работает нестабильно,
- servo могут дёргаться или не двигаться,
- возможны ложные срабатывания.

## Почему servo нельзя питать от 5V pin Arduino

Servo могут кратковременно брать большой ток (особенно при старте/упоре).

Если брать этот ток с `5V` Arduino pin:
- проседает питание Arduino,
- перезагружается контроллер,
- появляются ошибки по RFID,
- повышается риск повреждения платы/USB-порта.

Поэтому: servo питаются от **внешнего блока `POWER 5V`**.

## Роль 3.3V для MFRC522

MFRC522 — 3.3V модуль.

- Питание reader: только `POWER 3.3V`.
- Подавать 5V на питание reader нельзя.
- Если используете клоны модулей, всегда проверяйте маркировку платы и даташит.

## Варианты организации питания

### Вариант A (рекомендованный MVP)
- Raspberry Pi: отдельный штатный PSU.
- Arduino: через USB от Raspberry Pi.
- Servo: отдельный внешний `POWER 5V` PSU.
- MFRC522: от `3.3V` Arduino.
- Все GND объединены (`COMMON GND`).

Плюс: самый простой и безопасный старт.

### Вариант B
- Arduino питается от отдельного USB-адаптера.
- Raspberry Pi питается отдельно.
- Servo питаются отдельно.
- GND Arduino и servo PSU обязательно объединены.

Плюс: чуть меньше нагрузка на USB порты Raspberry Pi.

### Вариант C (позже, при росте узла)
- Выделенный мощный 5V PSU для servo-шины.
- Логика (Arduino/PCA9685) отдельно.
- Продуманная «звезда» GND и распределение линий.

Плюс: лучше масштабируется на много servo.

## Рекомендуемый вариант для MVP

Используйте **Вариант A**:

1. Raspberry Pi со своим официальным/качественным PSU.
2. Arduino подключён к Raspberry Pi по USB (`USB SERIAL` + питание Arduino).
3. Servo питаются от отдельного `POWER 5V` блока.
4. MFRC522 питается от Arduino `3.3V`.
5. Обязательно объединить `COMMON GND`:
   - GND Arduino
   - GND servo PSU
   - GND MFRC522
