# Power Design (Pico 2 W)

## Ключевой принцип

**Логика 3.3V (Pico + MFRC522) и силовая часть 5V (servo) должны быть разделены по питанию, но иметь common ground.**

## Рекомендованная схема для MVP

- Pico 2 W: питание по USB от Raspberry Pi.
- MFRC522: от 3.3V Pico.
- Servo: от отдельного внешнего 5V БП (минимум 3A для старта).
- Common ground: соединить GND Pico и GND servo PSU.

## Критично важно

- **Нельзя подавать 5V на GPIO Pico**.
- **Нельзя питать servo от Pico**.
- **Нельзя подключать 5V UART/TTL напрямую к Pico GPIO** без level shifting.
- Нельзя отлаживать всё сразу: сначала reader, затем servo, затем transport.

## Почему servo отдельно

- Пусковой ток servo может быть кратно выше номинала.
- Просадка питания вызывает reset Pico и/или нестабильный RFID.
- Разделение питания резко уменьшает хаотичное поведение.

## Raspberry Pi питание

- Использовать штатный БП для Raspberry Pi (обычно 5V 3A+).
- Не питать Pi от той же силовой линии с шумными servo без фильтрации.

## Ток и запас

Грубая оценка:
- SG90: до ~500-700mA на пиках.
- 2 servo одновременно: до ~1.4A пиково.
- Закладывать минимум x2 запас по току от расчётного среднего.

## Future power expansion

Для 4+ servo:
- отдельная 5V power bus;
- предохранители по веткам;
- распределительная плата;
- PCA9685 + внешнее питание.
